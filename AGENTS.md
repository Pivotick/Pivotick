# AGENTS.md

This file provides guidance to AI coding agents when working with code in this repository.

## Project Overview

Pivotick is a TypeScript graph visualization library built on D3.js. It renders interactive directed/undirected graphs with force-directed and tree layouts, providing UI components (sidebar, toolbar, context menus, tooltips) and user interactions (drag, zoom, selection). It ships as both ES module and UMD bundle.

## Build & Development Commands

```bash
npm run dev              # Vite dev server (serves src/main.ts demo page)
npm run build            # TypeScript check + Vite library build → dist/
npm run lint             # ESLint check
npm run lint_fix         # ESLint autofix
npm run vitepress:dev    # Documentation dev server
npm run docs:build       # TypeDoc + VitePress build
```

There is no test framework configured. Use `npm run build` to verify TypeScript correctness and `npm run lint` to check style compliance.

## Code Style

- **No semicolons** — ESLint enforced, do not add semicolons
- **Single quotes** — ESLint enforced, do not use double quotes in TypeScript/JavaScript
- **TypeScript strict mode** — `noUnusedLocals` and `noUnusedParameters` are enabled; do not leave unused variables or imports

## Architecture

### Entry Point & Exports

`src/index.ts` exports the `Graph` class (as default and as `Pivotick`), `Node`, and `Edge`. For UMD builds, `Node` and `Edge` are also attached as properties on `Graph`.

### Core Classes

- **`Graph`** (`src/Graph.ts`) — Main entry point. Takes a container element, graph data (nodes/edges), and options. Orchestrates renderer, simulation, and UI.
- **`Node`** (`src/Node.ts`) — Graph vertex with data, styling, and position.
- **`Edge`** (`src/Edge.ts`) — Connection between nodes with styling and markers.
- **`Simulation`** (`src/Simulation.ts`) — D3 force simulation wrapper. Can run in a Web Worker (`SimulationWorker.ts` / `SimulationWorkerWrapper.ts`).
- **`GraphRenderer`** (`src/GraphRenderer.ts`) — Abstract renderer interface.

### Renderers (`src/renderers/`)

- `svg/` — Primary SVG renderer (production-ready)
- `canvas/` — Experimental Canvas renderer

Factory pattern via `GraphRendererFactory.ts` selects the renderer.

### Interfaces (`src/interfaces/`)

Configuration is driven by the `GraphOptions` interface which contains nested options:
- `RendererOptions` — Styling, shapes, icons
- `SimulationOptions` — D3 force parameters
- `LayoutOptions` — Tree layout settings
- `InterractionCallbacks` — Event handlers (note: the spelling "Interraction" is the existing convention in this codebase, do not "fix" it)
- `GraphUI` — UI mode and component settings (`'full'`, `'viewer'`, `'static'`)

### Plugins (`src/plugins/`)

- `layout/` — Tree/hierarchical layout algorithm
- `d3Forces/` — Custom D3 force implementations
- `analytics/` — Graph analysis (DAG, cycle detection)

### UI Components (`src/ui/`)

- `UIManager.ts` — Coordinates all UI components
- `Notifier.ts` — User notification system
- `components/` — Sidebar, toolbar, context menus, tooltips

### Utilities (`src/utils/`)

- `ElementCreation.ts` — DOM element and safe ID generation
- `GeometryHelper.ts` — Geometric calculations for rendering
- `Getters.ts` / `GraphGetters.ts` — Data accessor helpers

### Styling (`src/styles/`)

SCSS with CSS custom properties for theming.

## Build Output

Vite library mode builds to `dist/` with two formats:
- `pivotick.js` (ES module)
- `pivotick.umd.cjs` (UMD)

Note: `vite.config.js` references `src/index.js` as entry — Vite resolves this to `src/index.ts`.

## Documentation

VitePress docs live in `docs/` with API docs auto-generated by TypeDoc into `docs/api/`. The docs site deploys to GitHub Pages via GitHub Actions.
